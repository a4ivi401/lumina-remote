# LuminaRemote: MVP Backlog & Roadmap

Этот документ содержит план разработки LuminaRemote, разделенный на фазы, с детализацией конкретных задач для разработчиков. Он будет использоваться для отслеживания прогресса.

## Фаза 1: Proof of Concept (PoC) — Основы транспорта и видео (Оценка: 4 недели)
**Цель:** Доказать работоспособность архитектуры, настроить фундамент сети и видео, замерить задержку (< 50 мс в локальной сети).

- [x] **TASK-1.1:** Инициализация проекта и базовой структуры (cargo workspace).
- [x] **TASK-1.2:** Написать модуль `lumina-core` (Криптография).
  - [x] Реализовать `derive_key_pair(seed: &str) -> (StaticSecret, PublicKey)`.
  - [x] Использовать `argon2` для деривации ключа из сид-фразы.
- [x] **TASK-1.3:** Настроить `quinn` (QUIC) в `lumina-network`.
  - [x] Интеграция генерации ключей как `rustls::CustomCertVerifier` / PSK.
  - [x] Разработка тестового обмена данными (echo-клиент/сервер в P2P).
- [x] **TASK-1.4:** Модуль захвата экрана `lumina-capture`.
  - [x] Использование кроссплатформенного крейта (`xcap`) для быстрого старта (PoC).
  - [ ] *Примечание:* На финальной стадии (Фаза 4) переписать на нативные API (DirectX Desktop Duplication, ScreenCaptureKit) для максимальной производительности.
- [x] **TASK-1.5:** Модуль кодирования видео `lumina-encoder`.
  - [x] Интеграция FFmpeg (`ffmpeg-sys-next`) для аппаратного кодирования в H.264 (Архитектурный каркас/Stub).
- [x] **TASK-1.6:** Модуль декодирования и рендеринга.
  - [x] Окно для тестов видеопотока (через `winit` + `softbuffer` или `egui`).
- [x] **TASK-1.7:** Передача ввода (`lumina-input`).
  - [x] Инъекция событий мыши и клавиатуры на стороне хоста (через `enigo` или кастомное решение).

## Фаза 2: Сетевой слой и Сигнализация (Оценка: 4 недели)
**Цель:** Полноценное P2P подключение через WAN с пробитием NAT и авторизацией.

- [x] **TASK-2.1:** Signal Server (`lumina-signal-server`).
  - [x] Реализация Actor-модели на `tokio`.
  - [x] Хранилище сессий in-memory (с опциональным Redis).
  - [x] REST API и WebSocket-эндпоинты для обмена IP-адресами.
- [x] **TASK-2.2:** Подключение Клиента к Signal Server.
  - [x] Логика регистрации `PublicKey` и поддержания сессии.
- [x] **TASK-2.3:** NAT Traversal (STUN).
  - [x] Обмен ICE-кандидатами и Hole Punching (пробитие NAT).
- [x] **TASK-2.4:** Базовый Relay Server.
  - [x] Сервер-посредник для проксирования QUIC трафика в случае Symmetric NAT.

## Фаза 3: Локальная сеть (LAN) и Адаптивность (Оценка: 3 недели)
**Цель:** Работа в любых сетях, мгновенное подключение в LAN, динамическая адаптация качества.

- [x] **TASK-3.1:** mDNS-обнаружение в `lumina-network`.
  - [x] Широковещательный поиск устройств в локальной сети.
- [x] **TASK-3.2:** Прямое подключение в LAN.
  - [x] Алгоритм пингов на локальные IP перед использованием WAN/Relay.
- [x] **TASK-3.3:** Adaptive Bitrate (QoS).
  - [x] Периодическая отправка метрик (RTT, Packet Loss, Decode Time).
  - [x] Динамическое изменение FPS, разрешения и качества (QP) на лету.
- [x] **TASK-3.4:** Оптимизация захвата экрана.
  - [x] Вычисление и отправка только "грязных прямоугольников" (Dirty Rects).
- [x] **TASK-3.5:** Обработка потери фокуса.
  - [x] Пауза захвата экрана и кодирования, если клиент свернул окно (экономия CPU/GPU).

## Фаза 4: GUI и Кроссплатформенность (Оценка: 5 недель)
**Цель:** Коммерческий внешний вид, UI-клиент (Tauri) и поддержка macOS/Linux.

- [ ] **TASK-4.1:** Инициализация проекта Tauri + React/TS.
  - [ ] Настройка фронтенд-проекта и интеграция с нашими Rust-библиотеками (core, network).
- [ ] **TASK-4.2:** Базовый UI/UX (Современный молодежный стиль).
  - [ ] Верстка главного экрана (Dark Mode, неон, закругления).
  - [ ] Отображение локального Device ID и поля для ввода удаленного.
- [ ] **TASK-4.3:** Окно авторизации и Unattended Access.
  - [ ] Логика генерации 12-символьного PIN-кода на Хосте.
  - [ ] Ввод PIN-кода на Клиенте и галочка "Сохранить машину" (безопасное локальное сохранение).
- [ ] **TASK-4.4:** Список "Мои машины".
  - [ ] Отображение сохраненных компьютеров в виде карточек с предпросмотром рабочего стола.
- [ ] **TASK-4.5:** Системный трей и фоновая работа.
  - [ ] Сворачивание в трей при закрытии на крестик, фоновая работа P2P сервера.
