# LuminaRemote: MVP Backlog та План розробки

Цей документ містить план розробки LuminaRemote, поділений на фази, з деталізацією завдань для розробників.

## Фаза 1: Proof of Concept (PoC) — Основи транспорту та відео (Оцінка: 4 тижні)
**Мета:** Довести працездатність архітектури, налаштувати мережу та відео, виміряти затримку (< 50 мс в LAN).

- [x] **TASK-1.1:** Ініціалізація проєкту та базової структури (cargo workspace).
- [x] **TASK-1.2:** Написати модуль `lumina-core` (Криптографія).
  - [x] Реалізувати `derive_key_pair(seed: &str) -> (StaticSecret, PublicKey)`.
  - [x] Використати `argon2` для деривації ключа з сід-фрази.
- [ ] **TASK-1.3:** Налаштувати `quinn` (QUIC) в `lumina-network`.
  - [ ] Інтеграція генерації ключів як `rustls::CustomCertVerifier` / PSK.
  - [ ] Тестовий обмін даними (echo-клієнт/сервер P2P).
- [x] **TASK-1.4:** Модуль захоплення екрана `lumina-capture`.
  - [x] Використання кросплатформного крейта (`xcap`) для швидкого старту (PoC).
  - [ ] *Примітка:* На фінальній стадії (Фаза 4) переписати на нативні API (DirectX Desktop Duplication, ScreenCaptureKit) для максимальної продуктивності.
- [ ] **TASK-1.5:** Модуль кодування відео `lumina-encoder`.
  - [ ] Інтеграція FFmpeg (`ffmpeg-sys-next`) для апаратного H.264 кодування.
- [ ] **TASK-1.6:** Модуль декодування та рендерингу.
  - [ ] Вікно для тестів відеопотоку (`winit` + `softbuffer` або `egui`).
- [ ] **TASK-1.7:** Передача вводу (`lumina-input`).
  - [ ] Ін'єкція подій миші та клавіатури на хості.

- [ ] **TASK-2.1:** Signal Server (`lumina-signal-server`).

  - [ ] Сховище сесій in-memory.
